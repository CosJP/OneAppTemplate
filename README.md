# 概要
One!Neighbor は、犬を飼っている人向けの交流アプリです。
主な利用者は犬を飼っている人ですが、犬を飼っていない人も閲覧目的で利用することを想定します。

# 機能
- 自分の飼っている犬を登録する機能
  - 写真
    - プロフィール写真の他に、任意で複数枚の写真を掲載可能とする
  - 名前
  - 犬種
  - 年齢
  - 体重
  - 使っているフード
  - 使っているサプリ
  - 自由記載の自己紹介(好きな食べ物、遊びなど)
  - お友達
    - 本アプリ上で登録済の他の犬とお友達として繋がることが可能
  - レスキュー機能
    - 犬が失踪などによって所在不明となった際に、自分の犬の迷子フラグを立てることでレスキュー機能が発動
    - 近所の同アプリユーザーに通知が飛ぶとともに、所在不明の犬の情報が通知される
    - レスキュー機能はユーザー本人のみが発動可能
  - 多頭飼い対応
    - 一人のユーザーが多頭飼いしている場合、複数の犬を登録可能

- タイムライン機能
  - ユーザーが増えてきたら将来的に実装を検討する
  - 一般的なSNSのように、メッセージや写真を投稿できる機能
  - 初期段階ではユーザー数が少ないこと、写真がストレージを逼迫することから実装を先延ばし

- お友達機能
  - 他のユーザーと友達になることができる
    - 他のユーザーのプロフィール画面から友達申請
  - 申請は承認、または拒否が可能

- ブロック機能
  - 他のユーザーをブロックする機能
  - ユーザープロフィール画面からブロックを設定可能

- おおよその位置情報を収集する機能
  - 収集する位置情報の範囲はGeohash Level 6(1.2km × 1.2km)
    - DBに保存するのはGPS座標をLevel 6に変換した文字列
    - 検索時に表示するのは〇km以内という表示のみ
      - 検索時にn km 以内のワンコを探す、みたいなことが可能
    - レスキューは Geohash Level 5 (約5km四方)で通知を送信
  - 位置情報の収集は任意とする
  - 位置情報を収集しない場合、近所の犬を検索する機能は使えない

- 近所の犬を検索する機能
  - 収集された位置情報に基づいて、近所の犬を検索可能

- 相談掲示板
  - 犬に関する悩みなどを書き込み、他の利用者と相談することが出来る掲示板
    - 特定の犬種でのフードの相性や体形ごとの運動量、しつけ方など
    - あくまで自己責任での利用を想定し、治療目的での利用はしないよう注意文を記載

- プッシュ通知機能
  - 以下の各項目ごとにプッシュ通知が可能、通知は項目ごとに無効化も可能
    - 近所の犬がレスキュー発動
    - DMを受信
    - 友達申請を受信
    - 近くにワンコがいます！
    - 相談掲示板に返信あり

- アカウント削除機能
  - ユーザーは自身の情報を削除することが可能
  - 削除時、DBから user_id に関するすべての情報を削除
  - S3 上からもコンテンツを削除
  - ただし、掲示板上では匿名化削除を行うので、board_posts, board_comments の user_id は NULL を許容する

- 通報機能
  - 不適切なユーザー、投稿、コメント、写真を通報する機能

- お引越し機能
  - 位置情報を再取得する

- 管理者機能
  - サービス運営管理者側から、サービスを管理するための機能
    - 掲示板のカテゴリ追加・削除
    - 通報された内容の確認
      - 承認
      - 取り下げ

# 懸念事項
- プライバシー、犯罪への悪用
  - 位置情報および飼っている犬の情報を収集するため、空き巣などの犯罪に悪用されるケースがある
    - 散歩などの時間は記載しないよう注意文を記載
    - 位置情報はかなり大まかなものを収集するようにする
- 日本の法律への対応
  - チャットなどの機能を有するアプリは届け出が必要

- ペットが亡くなったときの対応
  - 虹の橋を渡ったアーカイブ機能（プロフィール・写真を大切に残す）
  - アーカイブ時の通知自動オフ（誕生日通知とか辛いから）
  - 相談掲示板にペットロスカテゴリ（同じ痛みを経験した人と繋がれる場所）

# 対応OS
- 最初は Android 向けのアプリとして開発するが、将来的に iOS への対応も行う

# 利用料金
- 広告型による基本料金無料を想定

# 基盤
- AWS 上での構築
  - S3
    - 犬の画像や静的コンテンツの配置
  - CloudFront
  - ALB
  - ECS Fargate
  - Aurora RDS
    - PostgreSQL
  - SNS

# 認証
- ALB の OIDC 連携で認証
  - POST /api/v1/users のみ ALB 認証スキップ
  - バックエンドで JWT 署名検証 → user_identities で重複チェック → 登録

# Claude Codeとの相談結果
- DBに保存するもの: ユーザーの GPS 座標を Geohash Level 6 に変換した文字列（精確な住所はわからない）
  - 検索時に表示するもの: 「〇km以内」という距離のみ（座標は見せない）
  - レスキュー通知の範囲: Geohash Level 5（約5km四方）

- ├── 画像取得 → CloudFront → S3
  └── API通信 → ALB → ECS Fargate → Aurora
